### 2. 组织方式更优

**问题**: 为什么 JS 有 111 个测试用例，但 Rust 只有 3 个测试函数？

**答案**: Rust 测试将多个相关的断言合并到单个测试函数中，提高了效率。

#### 示例：bash.spec.js vs bash_spec.rs

**JavaScript** (111 个独立测试):
```javascript
describe('bash.spec', () => {
  it('"a/b/.x" should match "**/.x/**"', () => {
    assert(isMatch('a/b/.x', '**/.x/**', { bash: true }));
  });
  it('".x" should match "**/.x/**"', () => {
    assert(isMatch('.x', '**/.x/**', { bash: true }));
  });
  // ... 109 个更多的独立测试
});
```

**Rust** (3 个测试函数包含相同的 111 个断言):
```rust
#[test]
fn dotglob_tests() {
    let opts = bash_options();
    assert_is_match("a/b/.x", "**/.x/**", opts.clone(), true);
    assert_is_match(".x", "**/.x/**", opts.clone(), true);
    // ... 所有相关的断言在这里
}

#[test]
fn glob_tests() {
    // ... 更多断言
}

#[test]
fn globstar_tests() {
    // ... 更多断言
}
```

**结果**: ✅ **所有 111 个断言都存在**，只是组织成 3 个测试函数而不是 111 个。
