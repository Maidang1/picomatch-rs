### Implementation Notes

The Rust tests are generated by `scripts/extract_tests.py` at assertion granularity. Each `assert()` call in the JS tests maps to one `#[test]` in Rust.

Generator correctly handles:
- **JS string escape evaluation**: `\\[` in JS source → `\[` actual value → `"\\["` in Rust source (not double-escaped)
- **`windows` option**: 4 tests use `bash_options(false)` for `{ bash: true, windows: false }` cases (test_154, test_365, test_533, test_639)

### Re-verification 2026-03-17

Previous generator had two bugs that were fixed:
1. **Backslash doubling bug**: Generator read raw JS source text and doubled all `\` for Rust, instead of first evaluating JS escape sequences. This caused 26 tests to use doubled backslashes (e.g., `\\[` in actual value instead of `\[`). Fixed by adding `js_string_value()` in the generator.
2. **`windows` option always `true`**: Generator always used `bash_windows_options()` (windows=true) even for 4 tests that specify `{ windows: false }`. Fixed by generating `bash_options(windows)` with the correct boolean.

After the fix, all 649 tests were regenerated and re-verified to have string values that exactly match the JS originals.
